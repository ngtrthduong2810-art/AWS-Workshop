Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Bootstrapping the Cloud-Native Mindset](1.1-week1/)

**Week 2:** [Establishing Network Infrastructure and Access Control](1.2-week2/)

**Week 3:** [Mastering Compute Capacity and Server Operations](1.3-week3/)

**Week 4:** [Building Data Architecture and Application Tier Separation](1.4-week4/)

**Week 5:** [Shaping the FinOps Mindset in Software Development](1.5-week5/)

**Week 6:** [Shaping the Observability Mindset in Infrastructure Operations](1.6-week6/)

**Week 7:** [Building the Core Security and Network Foundation](1.7-week7/)

**Week 8:** [Shifting the Mindset from ClickOps to ScriptOps](1.8-week8/)

**Week 9:** [Reviewing the Architecture as a "Service User" Rather Than a "Code Writer"](1.9-week9/)

**Week 10:** [Laying the Serverless Architecture Foundation for InboxIQ](1.10-week10/)

**Week 11:** [Real-time System Implementation — API Gateway WebSocket and Gmail Integration in Worker](1.11-week11/)

**Week 12:** [The Week of "Invisible" Bugs — End-to-End Integration Debugging](1.12-week12/)
